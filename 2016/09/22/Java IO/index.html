Java IO

Veröffentlicht am 2016-09-22

基本IO

  1. java中字面量默认为int类型，并且byte、char、int任意两个类型相加的结果都为int类型。

  2. 不能将int类型的变量直接赋值给byte或char类型。但是byte b = 42是可以的，因为字面量是编译期常量，编译器会自动做类型转换。

    byte b1 = 32; 
byte b2 = 33;
byte b3 = b1 + b2;  // 结果是int类型变量，编译器会报错

  3. unicode是字符集 (2 bytes)，而不是字符的编码方式。

    • 代码点：可用于编码字符集的数字。

    • 代码单元：字符编码的基本单元，如UTF-16是2 bytes。

      JVM内部使用UTF-16编码方式，一个char实际上是一个UTF-16的代码单元，因此有一些字符需要使用两个char来表示。

  4. EOF字符在mac或unix平台下用ctrl-D表示，而在windows平台下用ctrl-Z表示。有时需要在单独的一行中输入这个字符。

  5. 几乎所有IO方法都会抛出IOException，但是PrintStream和PrintWriter类的方法除外，因为每次调用System.out.println()都用try/catch块包含会很不方便。

  6. System.in是InputStream的实例，System.out是PrintStream的实例。

  7. 重定向

    • 在shell中重定向：java OptimumBattingOrder < yankees06.tab > yankees06.out
    • 在程序内重定向：System.setIn()、System.setOut()、System.setErr()

OutputStream

OutputStream是一个抽象类，需要实现write(int b)方法。
